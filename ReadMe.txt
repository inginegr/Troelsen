Данное решение представляет магазин с товарами, расположенными в базе данных. 
Логика взаимодействия с базой данных выведена в отдельный .dll файл и реализована с помощью
Entity Framework и SQL комманд (два разных класса в dll). В проекте самого сайта можно поменять 
лигику, изменив в контейнере зависимостей (Ninject), соответствующую строку подключения.
Так же решение содержит тест, в котором запускается параллельно 1000 потоков, пытающихся получить доступ к 
базе данных (имитация большого количества покупателей). Данные потоки извлекают определенный товар,
пока он не закончится. Потоки, которые успешно извлекли некоторое колличество товара, заносятся в таблицу
базы данных.
